Campo de Código de Barras no Cadastro de Produtos

Schema / DB tRPC Router UI / Formulário

Adição de um campo opcional barcode ao cadastro de produtos, persistido no banco de dados e exposto via API tRPC. O campo aceita formatos como EAN-13, EAN-8 e Code 128.

Alterações por camada
Arquivo O que foi alterado
packages/db/src/schema.ts Coluna barcode varchar(50) adicionada à tabela products, nullable e sem valor padrão.
apps/web/src/lib/trpc/routers/products.ts barcode: z.string().nullable() adicionado ao productSchema de saída; barcode: z.string().max(50).optional() adicionado aos inputs de create e update.
apps/web/src/app/admin/products/page.tsx Campo barcode adicionado ao schema Zod do formulário, aos defaultValues, ao payload de submit, ao openEdit e ao JSX do dialog — com placeholder EAN13, EAN8, Code128....
apps/web/src/messages/en.ts
apps/web/src/messages/pt-BR.ts		 Chave barcode adicionada à seção products: "Barcode" / "Código de Barras".
Características
  • Campo opcional — produtos existentes não são afetados
  • Limite de 50 caracteres (cobre EAN-13, Code 128 e similares)
  • Renderizado abaixo do campo Category e acima da seção Fiscal Data
  • Schema migrado automaticamente via drizzle-kit push na inicialização do servidor

Busca Híbrida no Campo "Add Product" — Tela POS

Combobox cmdk filter POS Page

O campo Add Product na tela POS passou a suportar busca por nome e por código de barras simultaneamente, usando um filtro customizado no componente cmdk.

Como funciona
  • Cada item do Combobox recebe uma prop searchValue contendo "nome barcode"
  • A prop filter do Command (cmdk) substitui a busca fuzzy padrão por uma busca substring case-insensitive sobre searchValue
  • O texto exibido na lista continua sendo o name completo com preço e estoque
  • A separação entre dado de display e dado de busca é feita via value={item.searchValue} no CommandItem vs {item.name} no corpo do item
Alterações por arquivo
ArquivoAlteração
packages/ui/src/components/combobox.tsx Interface ComboboxItem ganhou prop opcional searchValue?: string; Command recebe filter={customFilter}; CommandItem usa value={item.searchValue ?? item.name}.
apps/web/src/app/admin/pos/page.tsx Items do Combobox de produtos passam searchValue: `${p.name} ${p.barcode ?? ""}`.trim().
Compatibilidade retroativa
  • Comboboxes de customers e payment methods não passam searchValue — a lógica cai para item.name (comportamento idêntico ao anterior)
  • O filtro customizado é aplicado apenas quando onScannerInput está presente (indicador de modo POS)

Detecção de Scanner via Timestamp

Combobox onKeyDown / timing POS Page

Leitores de código de barras emitem caracteres em rajada com intervalo muito curto entre teclas (< 30 ms) e finalizam com Enter. A detecção utiliza essa característica para diferenciar entrada de scanner de digitação manual sem necessidade de hardware especial ou APIs adicionais.

Algoritmo de detecção
// Refs persistem entre renders sem causar re-render
const lastKeyTimeRef = useRef<number>(0);
const intervalsRef   = useRef<number[]>([]);

// A cada tecla que NÃO é Enter:
const interval = Date.now() - lastKeyTimeRef.current;
intervalsRef.current.push(interval);   // acumula até 30 amostras

// Quando Enter é pressionado:
const avg = intervals.reduce((a, b) => a + b, 0) / intervals.length;

if (avg < 50 && code.length >= 4 && intervals.length >= 2) {
  // É um scanner → dispara onScannerInput e limpa o campo
  onScannerInput(code, quantity);
}
// Caso contrário → Enter normal (cmdk seleciona o item em foco)
Parâmetros de detecção
  • SCANNER_MAX_INTERVAL_MS = 50 — intervalo médio máximo entre teclas para ser considerado scanner
  • SCANNER_MIN_LENGTH = 4 — comprimento mínimo do código (evita falsos positivos com teclas soltas)
  • Mínimo de 2 amostras de intervalo coletadas antes de avaliar
  • Refs são reiniciadas ao fechar o popover e após cada detecção
Resolução no POS
  • O handleScannerInput busca em products pelo campo barcode com match exato
  • Se encontrado → chama handleSelectProduct(product.id, quantity)
  • Se não encontrado → exibe toast.error com o código lido

Multiplicador de Produto — Sintaxe N*busca

Combobox Parser / UX POS Page

O campo Add Product reconhece o caractere * como separador entre uma quantidade e um termo de busca (nome ou código de barras). O produto é adicionado diretamente com a quantidade especificada, sem etapas adicionais.

Exemplos de uso
"arroz"       → busca por nome, adiciona 1 unidade
"7891234"     → busca por barcode, adiciona 1 unidade
"3*arroz"     → busca por nome, adiciona 3 unidades
"2*7891234"   → busca por barcode, adiciona 2 unidades
"5*" + scanner → scan detectado após *, adiciona 5 unidades
Função parseMultiplier
function parseMultiplier(raw: string) {
  const starIndex = raw.indexOf("*");
  if (starIndex <= 0) return { quantity: 1, searchTerm: raw };

  const qty = parseInt(raw.slice(0, starIndex), 10);
  if (isNaN(qty) || qty <= 0) return { quantity: 1, searchTerm: raw };

  return { quantity: qty, searchTerm: raw.slice(starIndex + 1) };
}
Regras de validação
  • * deve ter um número inteiro positivo à esquerda — caso contrário é ignorado
  • "0*termo" e "abc*termo" são tratados como busca normal pelo input completo
  • O customFilter extrai automaticamente apenas o searchTerm (após *) para filtrar a lista
  • A detecção de scanner também opera sobre searchTerm, ignorando o prefixo de quantidade
Feedback visual
  • Quando quantity > 1, um banner aparece abaixo do CommandInput mostrando N× termo
  • O campo do input exibe o valor bruto ("3*arroz"), mantendo o fluxo natural de digitação
Validação de estoque com multiplicador
  • handleSelectProduct recebe quantity como segundo parâmetro (default 1)
  • Verifica: currentQty + quantity <= product.in_stock antes de adicionar
  • Produto já na lista → incrementa p.quantity + quantity
  • Produto novo → inicializa com quantity diretamente
🗺️

Fluxo Completo Unificado

Visão Geral
Input no CommandInput (campo Add Product) │ ├── Sem "*" │ ├── Digitação lenta → filtra lista por nome OU barcode → seleciona → qty = 1 │ └── Digitação rápida (scanner) → busca direta por barcode exato → qty = 1 │ └── Com "*" ex: "3*" ├── Parte após "*" digitada lentamente │ → filtra lista por nome OU barcode → seleciona → qty = 3 └── Parte após "*" digitada por scanner (rápida + Enter) → busca direta por barcode exato → qty = 3
