## Visão Geral

O serviço de notas fiscais orquestra o ciclo de vida completo de um documento fiscal eletrônico: desde a construção do XML até o envio ao SEFAZ, tratamento de respostas e persistência dos resultados. Ele fica na camada de aplicação/serviço, coordenando lógica de domínio, infraestrutura e persistência.

Esses três arquivos são a camada acoplada ao banco que fica na aplicação, não no pacote `@finopenpos/fiscal` (que tem zero dependências de banco de dados).

**Arquivos**: `apps/web/src/lib/invoice-service.ts`, `apps/web/src/lib/fiscal-settings-repository.ts`, `apps/web/src/lib/invoice-repository.ts`

## Ciclo de Vida da Nota Fiscal

```
  ┌──────────┐     ┌──────────┐     ┌───────────┐     ┌────────────┐
  │ pending   │────→│authorized│────→│ cancelled  │     │   voided   │
  └──────────┘     └──────────┘     └───────────┘     └────────────┘
       │                                                      ↑
       │           ┌──────────┐                               │
       ├──────────→│ rejected │                    (number range)
       │           └──────────┘
       │           ┌──────────┐
       ├──────────→│  denied  │
       │           └──────────┘
       │           ┌────────────┐
       └──────────→│contingency │──→ (sync later → authorized/rejected)
                   └────────────┘
```

## Operações Principais

### issueInvoice

```typescript
async function issueInvoice(
  orderId: number,
  model: InvoiceModel,  // 55 or 65
  userUid: string,
  recipientTaxId?: string,
  recipientName?: string
): Promise<{ invoiceId: number; status: InvoiceStatus; accessKey: string }>
```

**Fluxo**:
1. Carregar e validar configurações fiscais (`loadValidatedSettings`)
2. Verificar CSC para NFC-e modelo 65
3. Carregar pedido com itens do banco
4. Obter próximo número/série das configurações
5. Construir `InvoiceBuildData` a partir dos itens do pedido
6. `buildInvoiceXml()` → XML não assinado + chave de acesso
7. `loadCertificate()` + `signXml()` → XML assinado
8. Enviar ao SEFAZ via `sefazRequest()`
9. Parsear resposta → determinar status (autorizada/rejeitada/denegada)
10. Se autorizada: `attachProtocol()` → XML nfeProc
11. `saveInvoice()` → persistir no banco
12. `incrementNextNumber()` → atualizar contador

**Fallback offline** (apenas NFC-e): Se o SEFAZ estiver indisponível, salva com status "contingency" e sincroniza depois.

### checkSefazStatus

```typescript
async function checkSefazStatus(userUid: string): Promise<{
  online: boolean;
  statusCode: number;
  statusMessage: string;
}>
```

Chama `NfeStatusServico` para verificar se o SEFAZ está online. Status 107 = em operação.

### cancelInvoice

```typescript
async function cancelInvoice(
  invoiceId: number,
  userUid: string,
  reason: string  // min 15 chars
): Promise<{ success: boolean; statusCode: number }>
```

Apenas notas autorizadas podem ser canceladas. Constrói o XML do evento de cancelamento, assina e envia ao SEFAZ `RecepcaoEvento`. Salva o evento em `invoiceEvents`.

### voidNumberRange

```typescript
async function voidNumberRange(
  model: InvoiceModel,
  series: number,
  startNumber: number,
  endNumber: number,
  reason: string,
  userUid: string
): Promise<{ success: boolean; statusCode: number }>
```

Inutiliza faixas de números de notas não utilizadas (inutilização). Necessário quando números são pulados (ex.: falha do sistema antes da emissão).

### syncPendingInvoices

```typescript
async function syncPendingInvoices(userUid: string): Promise<{
  total: number; authorized: number; failed: number;
}>
```

Reenvia notas pendentes/em contingência que não foram confirmadas pelo SEFAZ.

## Validação de Configurações (DRY)

O helper `loadValidatedSettings` elimina a duplicação do padrão de validação repetido 4 vezes:

```typescript
async function loadValidatedSettings(userUid: string): Promise<FiscalSettings> {
  const settings = await loadFiscalSettings(userUid);
  if (!settings || !settings.certificatePfx || !settings.certificatePassword) {
    throw new Error("Fiscal settings or certificate not configured");
  }
  return settings;
}
```

Usado por: `checkSefazStatus`, `cancelInvoice`, `voidNumberRange`, `syncPendingInvoices`.

## Repositório de Configurações Fiscais (`apps/web/src/lib/fiscal-settings-repository.ts`)

### loadFiscalSettings

```typescript
async function loadFiscalSettings(userUid: string): Promise<FiscalSettings | null>
```

Mapeia campos snake\_case do banco para camelCase no domínio. Fornece valores padrão:
- Série: 1
- NCM: "00000000"
- CFOP: "5102" (venda interna)
- ICMS CST: "00", PIS/COFINS CST: "99"

### incrementNextNumber

```typescript
async function incrementNextNumber(userUid: string, model: InvoiceModel): Promise<void>
```

Incrementa atomicamente `next_nfe_number` ou `next_nfce_number`.

## Repositório de Notas Fiscais (`apps/web/src/lib/invoice-repository.ts`)

### Operações principais

| Função | Descrição |
|--------|-----------|
| `loadOrderWithItems(orderId, userUid)` | Carregar pedido + itens + produtos (para construção do XML) |
| `findInvoice(invoiceId, userUid)` | Obter nota fiscal por ID |
| `saveInvoice(data)` | Inserir nota + itens, retorna invoiceId |
| `findPendingInvoices(userUid)` | Obter notas com status "pending" ou "contingency" |
| `updateInvoiceStatus(invoiceId, data)` | Atualizar status, XML de resposta, protocolo, etc. |
| `saveVoidedInvoice(data)` | Criar registro "voided" para inutilização de faixa de números |
| `saveInvoiceEvent(data)` | Registrar evento de cancelamento/inutilização com XML |

### Multi-tenancy

Toda consulta inclui `eq(table.user_uid, userUid)` na cláusula WHERE.
