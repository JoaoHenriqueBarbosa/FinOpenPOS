## Visão Geral

SEFAZ (Secretaria da Fazenda) é o web service da autoridade tributária do Brasil. A comunicação utiliza SOAP 1.2 sobre HTTPS com TLS mútuo (certificado de cliente). Cada estado tem seu próprio SEFAZ ou delega ao SVRS (Sefaz Virtual do Rio Grande do Sul).

**Arquivos**: `sefaz-transport.ts`, `sefaz-request-builders.ts`, `sefaz-response-parsers.ts`, `sefaz-urls.ts`, `sefaz-event-types.ts`, `sefaz-reform-events.ts`, `sefaz-client.ts` (barrel)

## Arquitetura

```
invoice-service
    │
    ▼
sefaz-client.ts (barrel re-export)
    ├── sefaz-request-builders.ts  → builds XML request body
    ├── sefaz-transport.ts         → SOAP envelope + curl mTLS
    ├── sefaz-response-parsers.ts  → parses XML response
    └── sefaz-urls.ts              → resolves endpoint URLs
```

## Transporte (`sefaz-transport.ts`)

### Por que curl?

O `node:https` Agent do Bun não suporta PFX para mTLS (`ECONNREFUSED`). A solução alternativa: extrair cert/key PEM do PFX usando `certificate.ts`, gravar em arquivos temporários e chamar curl.

### sefazRequest()

```typescript
async function sefazRequest(options: {
  url: string;
  service: SefazService;
  xmlContent: string;
  pfx: Buffer;
  passphrase: string;
  timeout?: number;  // default 30s
}): Promise<{ httpStatus: number; body: string; content: string }>
```

**Fluxo**:
1. Construir envelope SOAP 1.2 encapsulando o conteúdo XML
2. Extrair cert/key PEM do PFX via `extractCertFromPfx()` / `extractKeyFromPfx()`
3. Gravar cert, key e corpo SOAP em arquivos temporários
4. `curl -s -k --cert ... --key ... -H "Content-Type: application/soap+xml" -d @body url`
5. Parsear status HTTP da saída `-w` do curl
6. Extrair conteúdo do corpo SOAP via regex
7. Limpar arquivos temporários no bloco `finally`

### Estrutura do envelope SOAP

```xml
<?xml version="1.0" encoding="UTF-8"?>
<soap12:Envelope xmlns:soap12="http://www.w3.org/2003/05/soap-envelope">
  <soap12:Header/>
  <soap12:Body>
    <nfeDadosMsg xmlns="http://www.portalfiscal.inf.br/wsdl/.../{service}">
      {xmlContent}
    </nfeDadosMsg>
  </soap12:Body>
</soap12:Envelope>
```

## URLs (`sefaz-urls.ts`)

### Mapeamento de autorizadores por estado

Cada estado brasileiro tem seu próprio SEFAZ ou delega ao SVRS:

| Autorizador próprio | Estados |
|---------------------|---------|
| AM | Amazonas |
| BA | Bahia |
| GO | Goiás |
| MG | Minas Gerais |
| MS | Mato Grosso do Sul |
| MT | Mato Grosso |
| PE | Pernambuco |
| PR | Paraná |
| RS | Rio Grande do Sul |
| SP | São Paulo |
| SVRS | Todos os outros 17 estados |

### Serviços

Cada autorizador expõe 6 web services:

| Serviço | Finalidade |
|---------|------------|
| `NfeStatusServico` | Verificar se o SEFAZ está online |
| `NfeAutorizacao` | Enviar NF-e para autorização |
| `NfeRetAutorizacao` | Recuperar resultado de autorização assíncrona |
| `NfeConsultaProtocolo` | Consultar nota pela chave de acesso |
| `NfeInutilizacao` | Inutilizar faixas de números não utilizadas |
| `RecepcaoEvento` | Enviar eventos (cancelamento, CCe, etc.) |

### getSefazUrl()

```typescript
getSefazUrl(stateCode: string, service: SefazService, environment: SefazEnvironment): string
```

Resolve a URL correta considerando estado, serviço e ambiente (produção vs homologação).

## Construtores de Requisição (`sefaz-request-builders.ts`)

### Verificação de status

```typescript
buildStatusRequestXml(stateCode, environment)
// → <consStatServ xmlns="..." versao="4.00"><tpAmb>2</tpAmb><cUF>35</cUF><xServ>STATUS</xServ></consStatServ>
```

### Autorização

```typescript
buildAuthorizationRequestXml(signedNfeXml, environment, stateCode)
// → <enviNFe xmlns="..." versao="4.00"><idLote>...</idLote><indSinc>1</indSinc>{nfe}</enviNFe>
```

### Cancelamento

```typescript
buildCancellationXml(accessKey, protocolNumber, reason, taxId, environment)
// Delegates to buildEventXml() with EVENT_TYPES.CANCELLATION
```

### Construtor genérico de eventos

```typescript
buildEventXml(options: {
  accessKey, eventType, sequenceNumber, taxId, orgCode,
  environment, eventDateTime, additionalTags?, lotId?
})
// → <envEvento>...<evento>...<infEvento>...<detEvento>...</detEvento></infEvento></evento></envEvento>
```

Construtores de nível superior delegam para `buildEventXml`: CCe, Ator Interessado, Pedido de Prorrogação, Comprovante de Entrega, etc.

### Construtores em lote

- `buildBatchManifestationXml()` — múltiplos eventos de manifestação em um envelope
- `buildBatchEventXml()` — eventos em lote genéricos

### Primitivas compartilhadas (`sefaz-event-types.ts`)

```typescript
buildEventId(eventType, accessKey, seqNum)  // → "ID110111{chNFe}01"
defaultLotId(lotId?)                         // → lotId ?? Date.now().toString()
getEventDescription(eventType)               // → "Cancelamento", "EPEC", etc.
```

## Parsers de Resposta (`sefaz-response-parsers.ts`)

```typescript
parseStatusResponse(xml)         // → { statusCode, statusMessage, averageTime }
```

## Eventos da Reforma (`sefaz-reform-events.ts`)

Eventos da reforma tributária IBS/CBS (PL\_010). Estes são distintos dos eventos padrão da NF-e.

### Padrão de template (DRY)

Todos os 14 eventos da reforma seguem a mesma estrutura:

```typescript
function buildXxx(config, chNFe, nSeqEvento, ...params, verAplic?) {
  const va = resolveVerAplic(verAplic, config.verAplic);
  let tagAdic = reformEventHeader(config, va, tpAutor);  // <cOrgaoAutor>+<tpAutor>+<verAplic>
  // ... event-specific content added to tagAdic ...
  return wrapReformEvent(config, tpEvento, chNFe, nSeqEvento, tagAdic, va);
}
```
